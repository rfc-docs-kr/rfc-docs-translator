## **1.  The SNMP Network Management Framework**

SNMP 네트워크 관리 프레임워크는 현재 이러한 구성 요소로 구성되어 있습니다. 그들은:

o RFC 1902\[2\]에 설명된 SMI - 관리 목적으로 개체를 설명하고 이름을 지정하는 데 사용되는 메커니즘입니다.

o SNMPv2에 대한 RFC 1903\[3\]에 설명된 텍스트 규칙.

o SNMPv2에 대한 RFC 1904\[4\]에 설명된 적합성 선언문.

o 에 설명된 단순 네트워크 관리 프로토콜

- RFC 1157 \[5\].

o RFC 1905\[6\]에 설명된 프로토콜 작업

-SNMPv2.

o MIB-II, STD 17, RFC 1213\[7\] - SNMPv2용 인터넷 프로토콜 제품군을 위한 관리 객체의 핵심 집합.

프레임워크는 실험 및 평가 목적으로 새 개체를 정의할 수 있도록 허용합니다.

### **1.1.  Object Definitions**

관리 객체는 MIB\(Management Information Base\)라고 하는 가상 정보 저장소를 통해 액세스됩니다. MIB의 개체는 SMI에 정의된 ASN.1\(Abstract Syntax Notation One\)의 하위 집합을 사용하여 정의됩니다. 특히 각 개체 유형은 관리상 할당된 이름인 OBJECT IDENTIFIER로 명명됩니다. 개체 인스턴스와 함께 개체 유형은 개체의 특정 인스턴스화를 고유하게 식별하는 역할을 합니다. 인간의 편의를 위해 객체 유형을 참조하기 위해 디스크립터라고 하는 텍스트 문자열을 자주 사용합니다.

## **2.  Overview**

이 MARS MIB는 RFC2022\[1\]에 설명된 대로 MARS 클라이언트, 서버 및 MCS\(멀티캐스트 서버\)를 관리하는 데 사용할 수 있는 관리 개체를 정의하도록 설계되었습니다. MIB는 하나 이상의 MARS 클라이언트가 실행 중이거나 하나 이상의 MARS 서버가 실행 중이거나 하나 이상의 MARS 멀티캐스트 서버가 실행 중인 시스템에서 사용해야 합니다.

이 MIB 모듈 정의에서는 \[1\]에 정의된 MARS에 대한 이해를 가정합니다. 그러나 다음 용어는 자주 사용되며 참조용으로 여기에 포함됩니다.

```text
   Multicast Group
```

한 엔드포인트에서 보낸 패킷을 멀티캐스트 그룹의 다른 모든 구성원이 수신하도록 서로 통신하는 엔드포인트 그룹입니다.

```text
   Multicast Address Resolution Server (MARS)
```

멀티캐스트 그룹 구성원 정보를 끝점에 배포하는 서버입니다.

```text
   Client/Endpoint
```

MARS에 등록하고 하나 이상의 멀티캐스트 그룹의 구성원인 ATM 연결 호스트 또는 라우터입니다. 엔드포인트는 다른 그룹 구성원에 대한 ATM 가상 채널\(VC\)을 설정하거나 멀티캐스트 서버를 사용할 수 있습니다.

```text
   Cluster
```

MARS에서 관리하는 클라이언트 집합입니다.

```text
   Multicast Server (MCS)
```

멀티캐스트 그룹의 엔드포인트 간에 ATM 가상 채널\(VC\)을 설정하고 엔드포인트가 대신 전송하기 위해 데이터 트래픽을 전달하는 서버입니다.

MIB는 MARS 클라이언트 그룹, MARS\(서버\) 그룹 및 MARS 멀티캐스트 서버\(MCS\) 그룹의 세 가지 주요 그룹으로 나뉩니다.

### **2.1.  The MARS Client Group**

이 클라이언트 그룹은 MARS 클라이언트 관리를 위해 MIB에서 구현해야 하는 개체 모음을 정의합니다. 여기에는 다음 테이블이 포함되어 있습니다.

```text

```

ATM 주소, 기본 MARS의 ATM 주소, 등록 상태 및 타이머와 같은 클라이언트에 대한 정보입니다.

```text
   o    MARS Client Multicast Group Table
```

MARS 클라이언트와 연결된 IP 멀티캐스트 주소 블록 목록입니다.

```text
   o    MARS Client Backup MARS Group Table
```

MARS 클라이언트와 연결된 백업 MARS 목록입니다.

```text
   o    MARS Client VC Table
```

클라이언트가 연 VC에 대한 정보.

```text
   o    MARS Client Statistics Table
```

MARS 클라이언트에서 수집한 통계입니다.

## **6.  Security Considerations**

이 MIB에는 읽기-쓰기 및/또는 읽기-만들기의 MAX-ACCESS 절이 있는 많은 관리 개체가 정의되어 있습니다. 이러한 개체는 일부 네트워크 환경에서 민감하거나 취약한 것으로 간주될 수 있습니다. 적절한 보호 없이 비보안 환경에서 SET 작업을 지원하면 네트워크 작업에 부정적인 영향을 미칠 수 있습니다.

SNMPv1 자체는 안전하지 않은 환경입니다. 네트워크 자체가 안전하더라도\(예: IPSec 사용\) 보안 네트워크에서 누가 이 MIB의 개체에 액세스하고 설정\(변경/생성/삭제\)할 수 있는지에 대한 제어가 없습니다.

구현자는 SNMPv3 프레임워크에서 제공하는 보안 기능을 고려하는 것이 좋습니다. 특히, 사용자 기반 보안 모델 RFC 2274\[8\] 및 뷰 기반 액세스 제어 모델 RFC 2275\[9\]의 사용을 권장합니다.

그런 다음 이 MIB에 대한 액세스 권한을 부여하는 SNMP 엔터티가 실제로 SET\(변경/생성/삭제 \) 그들을.

참고: 실제로 읽기 액세스에는 액세스 제어가 필요할 수도 있습니다.
