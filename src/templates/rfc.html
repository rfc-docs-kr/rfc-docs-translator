공지 : 일부 번역되지 않은 콘텐츠에 관하여


이번에 만들게된 번역 서비스를 공개하고 나니 google 번역 api 비용이 생각 보다 많이 청구 되었습니다.
충분히 검토 하지 못하고 마음만 앞서서 여기저기에 홍보한 결과라 생각 하고 반성 중입니다.

기존의 해당 서비스의 구조가

rfc 문서 -> 영역별 컨텐츠 식별 -> 번역 가능한 컨텐츠의 전처리(줄바꿈 및 공백 제거) 
                               -> 번역 api를 통한 번역 -> 마크다운으로 변환

이러한 흐름으로 기능이 작성 되었다면 

rfc 문서 -> 영역별 컨텐츠 식별 -> 번역 가능한 컨텐츠의 전처리(줄바꿈 및 공백 제거) 
                               -> 마크다운으로 변환 -> 브라우저 번역 기능으로 번역

위의 방식으로 변경이 되었습니다.

번역 api비용 문제를 해결 하기 전까지는 브라우저의 자체 번역기능으로 번역을 이용 해주시면 감사하겠습니다.

브라우저 별로 번역 품질이 달라질 수 있으나, 기존 번역 방식에 비해서는 품질이 크게 저하되지 않을 것으로 보입니다.
